## Formal Experiment Pattern

1. Start from `official-k562-gap-closing`.
2. Create a dedicated run branch for the experiment.
3. Use `scripts/run_official_k562_harness_search.py` or a prompt generated by `scripts/generate_codex_experiment_prompt.py`.
4. Keep strict artifacts on.
5. If `acquisition_queue.json` appears, run active artifact acquisition before reporting a blocker.
6. Commit only code, configs, docs, summaries, queues, proposals, and small audit reports.

## Current Hard Rule

Do not stop at a missing artifact queue without attempting acquisition. Missing artifacts mean:

```text
queue -> resolver -> ACQUIRE task -> source-backed search/build -> audit -> registry update -> resume
```

Only a failed acquisition attempt becomes a blocker.
